# Week 11 個人工作坊反思

### 1. 第一次自己決定客戶的感受
這週做為老闆第一次自己決定客戶是誰，我發現最卡的地方在於「第三格：他真正想要的是什麼」中的隱性需求。一開始很容易流於表面，覺得文案小編來找我就是想提升產量、學會下 Prompt 罷了。但當我試著從他的角度思考焦慮與恐懼時，才意識到他真正害怕的是被時代淘汰、在主管面前的廉價感。卡住時，我透過扣緊我們 ReBoot-Edge 「拒絕巨嬰、重構生存技能」的強烈硬派公司風格，回頭去想他需要的不是被摸頭安慰，而是重新奪回生活掌控感的底氣。

### 2. 從 Gemini CLI 換到 agy 之後，公司還活著的意義
當我發現把工具換到 Antigravity CLI (agy) 之後，公司完全不需要修改任何一行 spec.md 就能照常跑出如此驚豔的服務方案，這對我來說是一個很大的震撼。這意味著工具、平台或 CLI 都只是執行層面的螺絲釘，隨時可以被汰換；這間公司的真正核心資產與靈魂，是我在 spec.md 裡幫它建立的世界觀、知識主體性與 Specialists 協作邏輯。這給了我做為一個軟體架構師與創業者極大的底氣：我不綁定任何工具，工具是為我的商業邏輯服務的。
